# Trigger / Routing 设计说明

## 当前原则

- 触发层不再依赖本地关键词表决定是否“旁听接话”。
- 本地触发只保留结构信号：`@bot`、私聊、reply、活跃会话窗口、群热度。
- 具体要不要处理，交给 `Router + Agent` 做 AI 判定，不再走固定关键词命中。

## 本次调整

已移除以下本地关键词旁听路径：

- `ai_listen_keyword_enable`
- `ai_listen_keywords`
- `_count_listen_keyword_hits()`
- `ai_listen_probe_keyword`

## 当前旁听触发来源

- 明确结构输入：`@`、私聊、reply。
- 会话延续：最近 bot 刚回复过，且还在 follow-up window 内。
- 群热度：消息数和活跃人数达到阈值。
- 显式结构请求：控制 token、链接、文件扩展名、命令式输入等。

## 这样做的原因

- 降低“碰巧命中关键词就乱接话”的误触发。
- 避免把提示词和本地词表绑死，保持纯 AI 路由的一致性。
- 让测试更稳定：结构信号比自然语言关键词更容易验证。

## 后续规则

- 新增触发能力时，优先加结构信号，不加自然语言关键词表。
- 如果必须引入本地判定，必须证明无法用结构信号或 AI 路由替代。
